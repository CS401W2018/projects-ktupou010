Explain the difference between an absolute link and a relative link. Why would you use relative links?

An absolute link is a complete URL that includes the protocol (http/https), domain name, and path to the resource. A relative link is a URL that is relative to the current document or website, and does not include the protocol or domain name.

Relative links are useful when:

  • You're linking to another page within the same website.
  • You want to make it easy to move your website to a different domain or server.
  • You want to reduce the size of your HTML code.

Relative Link:

Structural Engagement

List the 3 rules with relative links:

  1. Same directory: Use the filename only (e.g., example.html).
  2. Subdirectory: Use the subdirectory name and filename (e.g., subdirectory/example.html).
  3. Parent directory: Use ../ to go up one level (e.g., ../example.html).

Do links always need to be inside a NAV tag?

No, links do not always need to be inside a NAV tag. Links can be used anywhere in an HTML document, and can be wrapped in various elements such as P, DIV, SPAN, or A. The NAV tag is specifically used for navigation menus.
